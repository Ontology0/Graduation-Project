# exp2 — A/B/C 타입 분해

## 목적
exp1에서 Base가 거짓 문서를 따라간 이유가 "품질 판단 실패"인지
"정답 문서가 아예 없어서"인지 분리 측정.

## 설계
- 모델: claude-haiku-4-5 (LiteLLM)
- 같은 사실 12개를 3가지 문서 구성으로 변환 → 36케이스

| 타입 | 문서 구성 | 측정 |
|---|---|---|
| A | 거짓 + distractor (정답 문서 없음) | 거짓 거부율 |
| B | 거짓 + 진짜 + distractor | 품질 판단 능력 (핵심) |
| C | 진짜 + distractor | 문서 활용율 (상한) |

## 결과
| 타입 | Base | Conflict-aware |
|---|---|---|
| A (거짓만) | 50% | 83% |
| B (거짓+진짜) | 100% | 100% |
| C (진짜만) | 100% | 100% |

## 해석
- **B/C에서 두 arm 모두 100%**: 진짜 문서가 함께 있으면 Base도 완벽히
  진짜를 골라냄 → "품질 판단 능력" 자체는 Base에도 있음.
- **A에서만 차이(50→83%)**: 정답 문서가 없고 거짓만 있을 때,
  conflict-aware가 자기 지식으로 거짓을 거부하는 능력이 큼.
- 즉 conflict-aware의 가치는 **"검색이 실패했을 때의 robustness"**에 한정.

## RQ와의 불일치 (중요)
이 실험은 "외부 거짓 vs 내부 정답" 구조다. 그러나 본 연구의 메인 RQ는
"내부 옛 지식 vs 외부 최신 지식"으로 **방향이 반대**다.
이 깨달음으로 exp3(temporal)로 재설계함.

또한 A 타입에서 conflict-aware도 실패한 2케이스는 모두 model_knows=False
(모델이 정확한 값을 모르는 신규 정보)였다. → parametric 지식이 없으면
프롬프트로도 거짓을 거부할 수 없다는 경계가 드러남.
